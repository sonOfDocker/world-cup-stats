# World Cup Dataset Analysis

## Overview
This document provides a detailed analysis of the Kaggle FIFA World Cup dataset (1930–2022). It serves as the foundation for the data ingestion pipeline and canonical model mapping.

## Discovered Files
The following dataset files were identified in `data/raw/kaggle/`:
- `fifa_world_cup_1930_2022_all_matches.csv`: Primary dataset containing 964 matches from 1930 to 2022.

## Data Quality Observations
- **Encoding:** The `Score` column often contains a special dash character (e.g., `4–1` instead of `4-1`).
- **Date/Time Formats:** `Match Date` uses `M/D/YYYY` format. `Match Time` is in `HH:mm`. Normalization to ISO-8601 is required for the canonical model.
- **Boolean-like Columns:** Columns like `Extra Time` and `Penalty Shootout` use `1` for true and `0` for false.
- **Penalty Shootout Metadata:** `Score Penalties` is often `0-0` even when no shootout occurred. The `Penalty Shootout` flag must be the primary driver for this logic.
- **Inconsistent Casing:** Header names like `tournament Name` have mixed casing.

## Mismatches with Documentation
- **Column Names:** `docs/data-dictionary.md` mentions `Key Id`, while the actual CSV header is `Key Id`. (Note: Minor casing variations in documentation vs CSV).
- **Canonical Match Identifier:** `docs/canonical/match.md` suggests an opaque stable identifier. The CSV provides a `Match Id` (e.g., `M-1930-01`) which is stable but source-specific. The implementation should likely map this to a `sourceId` field rather than using it as the primary `match_id` directly if a truly opaque internal ID is preferred.

## Impact on Canonical Model (Story #17)
1. **Normalization:** Ingestion must handle the `M/D/YYYY` date format and combine it with `Match Time` into a single ISO-8601 `kickoff_datetime`.
2. **Venue Identification:** A `venue_id` must be generated by a composite key of `(Stadium Name, City Name, Country Name)` to avoid duplicates and handle stadium name changes over time.
3. **Team Identification:** `team_id` should be derived from normalized `Home Team Name` and `Away Team Name`.
4. **Extra Time/Penalties:** The `home_goals` and `away_goals` in the canonical model must include extra time goals, which matches the CSV's `Home Team Score` and `Away Team Score` behavior.

## Conclusion
The dataset is highly structured and provides all necessary fields for the v1 canonical model. The primary challenges are date/time normalization and ensuring stable internal identifiers for teams and venues based on their names.
